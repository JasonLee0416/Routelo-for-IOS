# RouteLO 출시·수익화 전략 (GTM)

> 초기 베타를 유료로 낼 수는 없다. **베타의 임무는 "돈 버는 것"이 아니라, 나중에
> 돈 받을 자격(리텐션·PMF·지불의사)을 데이터로 증명하는 것**이다. 이 문서는 그
> 증명을 거쳐 수익화로 넘어가는 단계별 로드맵이다.

대상: 화환·화분 배송 기사(니치 B2B 유틸리티) · 시장: 한국 · 포지셔닝: 프라이버시
우선, "일 아껴주는 도구". 광고보다 **프리미엄/구독**이 정석(ADS 검토 결론은
본 문서 말미 참조).

---

## 0. 핵심 원칙 (안티패턴 먼저)
- ❌ 베타에서 유료화/광고 → 유입 퍼널·학습 데이터 다 죽인다.
- ❌ 기능 다 만들고 한 번에 유료 전환 → 근거 없는 가격은 반드시 틀린다.
- ✅ **무료로 사랑받고 → 지불의사 신호 확인 → 관대한 프리미엄으로 전환**.
- ✅ 베타 유저는 **평생 특별대우(그랜드파더링)** 로 전도사(evangelist)로 만든다.

---

## 1. 단계별 로드맵

### Phase 0 — Private Beta (지금 · TestFlight · 지인)
- **목표**: 핵심 가치 검증(“OCR이 진짜 시간을 아껴주나? 계속 쓰나?”), 정확도 개선.
- **가격**: 100% 무료. 광고·페이월 없음.
- **레버**: 방금 붙인 **옵트인 품질 리포트**로 실사용 교정 데이터 축적 → 정확도 개선 루프.
- **성공 지표(Exit criteria)**: 아래 §4의 활성화·리텐션이 "꺾이지 않고 평탄화"되기 시작.
- **행동**: 주 1회 유저 인터뷰(“없으면 불편한가? 얼마면 낼 건가?”), 버그·정확도 최우선.

### Phase 1 — Open Beta / 정식 출시 (무료)
- **목표**: App Store 공개, 유입 퍼널·리텐션 커브를 실제 규모로 확인.
- **가격**: 전원 무료. 단, **"파운딩 멤버" 포지셔닝** 시작 — 초기 유저에게 나중에
  Pro 혜택(평생 할인 또는 N개월 무료)을 약속(신뢰+희소성).
- **레버**: 앱스토어 최적화(ASO) — 이름/부제/키워드(이미 `APP_STORE_METADATA.md`),
  스크린샷 스토리, 초기 리뷰 확보. 배송기사 커뮤니티·화원 오프라인 채널 시딩.
- **동시 준비**: Pro 기능을 **페이월 플래그 뒤에 만들어두되 과금은 OFF**.

### Phase 2 — 수익화 (프리미엄/구독) — *PMF 신호 확인 후에만*
- **트리거(정량)**: §5의 유료화 진입 조건 충족 시 시작.
- **모델**: 관대한 **무료 티어 + Pro 구독(월+연, 연간 할인)**. 선불 유료 앱 ❌.
- **그랜드파더링**: 베타/파운딩 유저는 Pro를 일정기간 무료 또는 평생 할인 → 이탈·반감 방지.
- **가격**: 가치기반(아껴준 시간·기사 수입에 앵커링). 인터뷰+가격민감도 조사로 결정,
  이후 A/B. (초기 가설: ₩4,900–9,900/월, 연간 2개월치 할인)

### Phase 3 — 확장
- 추천(리퍼럴) 프로그램, **B2B 시트 판매**(화원·배송대행이 기사 계정 구매),
  팀/플릿 티어, 지역·차종 확장.

---

## 2. 무료 vs Pro 기능 분리 (가설 — 검증 대상)
원칙: **무료 티어도 "혼자서 충분히 쓸 만"** 해야 입소문이 난다. Pro는 "돈 버는 헤비
유저가 시간을 더 아끼는" 곳에 건다.

| 영역 | 무료 | **Pro (구독)** |
|---|---|---|
| 인수증 스캔 | 하루 N건(예: 5~10) | **무제한** |
| 배달 관리·동선·알림 | 기본 | 고급(시간창 최적화 등) |
| 차량 | 1대 | **다중 차량** |
| 수익/운영비 리포트 | 요약 | **상세 리포트·CSV 내보내기** |
| OCR 품질 | 온디바이스 | **클라우드 폴백(동의) 우선 처리** |
| 백업/동기화 | 로컬 | **클라우드 백업·기기 간 동기화** |

> 무료 한도(스캔 수 등)는 **가치는 주되 헤비유저는 벽을 느끼는** 선으로 조정(데이터로 튜닝).

---

## 3. 파운딩 멤버 / 그랜드파더링 (신뢰 플레이)
- 지금 베타 유저에게 **명시적으로 약속**: “정식 출시 후에도 당신은 Pro를 [평생 무료 /
  50% 평생 할인] 드립니다.” → 지금 이탈 방지 + 리뷰·추천 동력.
- 파운딩 멤버 뱃지/한정 혜택으로 소속감 부여.
- 유료화 시점에 **기존 무료 사용자에게 충분한 유예·안내**(갑작스런 벽 ❌).

---

## 4. 측정 지표 (이걸로 의사결정)
- **North Star**: 주간 활성 유저당 "관리한 배송/스캔 수"(가치 실현의 대리지표).
- **활성화(Activation)**: 신규가 첫 세션/첫날에 **스캔+등록 1회** 완료한 비율.
- **리텐션**: D1 / D7 / D30, WAU/MAU. 커브가 **평탄화**되면 PMF 신호.
- **핵심 루프 빈도**: 유저당 주간 스캔 수(늘면 습관화).
- **정성**: 인터뷰 + NPS. "없으면 불편?"에 40%+ "매우 불편"이면 강한 신호.
- **(유료화 후)** 무료→유료 전환율, 트라이얼→유료, MRR, 이탈률, LTV.
- 도구: 방금 만든 **텔레메트리를 지표 대시보드로 확장**(활성화/리텐션 이벤트 추가).

---

## 5. 유료화 진입 조건 (언제 과금 시작하나)
아래를 **동시에** 만족할 때 Phase 2 시작:
1. 리텐션 커브가 꺾이지 않고 평탄화(예: D30 리텐션이 목표선 이상 유지),
2. 주간 재사용하는 "찐 유저" 코호트가 유의미하게 존재,
3. 인터뷰/설문에서 **지불의사 확인**(가격민감도 조사 통과),
4. 유료로 잠글 **"헤비유저가 부딪히는 명확한 벽"** 기능이 존재.
> 하나라도 미달이면 무료 유지 + 그 지표를 끌어올리는 데 집중.

**저비용 검증 기법**
- **가짜문(fake-door)**: “Pro (곧 출시)” 버튼을 두고 탭률 측정 → 관심도 정량화.
- **가격민감도(Van Westendorp)**: 베타 유저에게 4문항 설문으로 수용가 범위 도출.
- **동시 정성**: 실제로 지갑 여는지(“지금 결제되면 하실래요?”)로 교차확인.

## 6. 기술·운영 스택 (수익화 시)
- **결제**: StoreKit2 기반 구독. 직접 서버 영수증검증 대신 **RevenueCat** 권장
  (구독/엔타이틀먼트/영수증 처리 대행, 소규모 매출 구간 무료). bare RN엔
  `react-native-purchases` 추가(네이티브 → pod 재빌드).
- **한국 현실 체크**: App Store로 매출 발생 시 **사업자등록·세금·유료계약 동의** 필요.
  → 지금 무료로 두는 것이 규제/세무 관점에서도 합리적(런웨이 확보).
- **개인정보**: 광고 SDK 미도입으로 현재 "추적 없음" 라벨 유지(강점). 구독은 결제
  데이터만 애플이 처리 → 프라이버시 포지셔닝 훼손 없음.

---

## 7. 광고에 대한 결론 (요약)
- 일반 무료앱엔 광고가 보편적이나, **업무용 도구엔 부적합**: 워크플로 방해, ATT/추적으로
  프라이버시 라벨 훼손, 니치 규모라 수익 미미.
- 정 필요하면 **비개인화 배너(설정/기록 화면) + "광고 제거" IAP** 최소화. 그러나
  본 앱은 **구독이 정석**.
