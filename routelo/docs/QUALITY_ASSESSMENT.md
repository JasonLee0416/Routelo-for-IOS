# RouteLO 품질 평가 (공식 표준 기준)

앱 성능·기능을 감이 아니라 **국제표준/공식 기준**으로 냉정하게 채점하고, 각 기능이
어느 수준인지 파악한 결과. 이를 근거로 "무료로 반드시 보완할 것" vs "Pro로 묶을 것"을
가른다. (평가일 기준 코드베이스 실측 근거 포함)

## 사용한 공식 심사 기준 (출처)
| 표준 | 무엇을 재나 | 성격 |
|---|---|---|
| **ISO/IEC 25010** (SQuaRE 제품 품질 모델) | 소프트웨어 제품 품질 8특성 | 국제표준 — 본 채점의 뼈대 |
| **Apple App Store Review Guidelines** | 심사 통과 요건(안전/성능/디자인/법무) | 애플 공식(합/불) |
| **Apple HIG** | 디자인·상호작용 품질 | 애플 공식 |
| **Apple 성능 지표**(MetricKit / Xcode Organizer) | 실행시간·행(hang)·메모리·에너지·크래시프리 | 애플 공식 계측 |
| **OWASP MASVS** | 모바일 앱 보안 검증(L1/L2) | 업계 표준 |
| **WCAG 2.2 / Apple 접근성** | 접근성(VoiceOver·Dynamic Type 등) | 국제/애플 표준 |

> 수치 임계값(예: 콜드 실행 <400ms, hang >250ms)은 애플 가이드의 통용 기준이며, 정밀
> 최신값은 딥리서치로 보강 가능.

## ISO/IEC 25010 스코어카드 (5점 척도)
근거는 이번 세션 실측: 테스트 47스위트/317+개, `index.tsx` 5,138줄, 접근성 프로퍼티 0개,
보안 설정 전부 OFF, OCR 정답률 26/31(≈84%), OCR `processingMs` 계측됨.

| # | 특성(ISO 25010) | 점수 | 근거 | 핵심 갭 |
|---|---|:--:|---|---|
| 1 | **기능 적합성** Functional Suitability | **3.5** | 배달·OCR·동선·알림·수익 핵심 완비, 실패 시 수동 폴백 | OCR 84%→상향, 미완 기능(차량/리포트/백업) |
| 2 | **성능 효율성** Performance Efficiency | **3.0** | 온디바이스 OCR·로컬 우선(네트워크 지연 0), 처리시간 계측 | 실행시간/hang/메모리 **실측 부재**(MetricKit 미도입) |
| 3 | **호환성** Compatibility | **3.0** | 내비 3종 딥링크, 카메라/사진 연동 | 좌표 미지오코딩→경로 정밀도 제한, iOS 전용 |
| 4 | **사용성** Usability | **2.5** | HIG 반영(아이콘·세이프에어리어·폰트 스케일·스와이프), 온보딩 | **접근성 0**(VoiceOver/Dynamic Type 미대응) — 큰 결함 |
| 5 | **신뢰성** Reliability | **3.5** | 로컬 우선·유실방지 큐·47 테스트 스위트·사진 상대경로 복원 | **크래시 리포팅 부재**(크래시프리율 미계측) |
| 6 | **보안** Security | **2.5** | 추적 없음·텔레메트리 PII 마스킹·프라이버시 우선 | **저장 암호화·앱잠금·생체 전부 미구현**(민감 PII 평문 로컬) |
| 7 | **유지보수성** Maintainability | **3.5** | services/domain 분리·테스트성 우수·CI | `index.tsx` **5,138줄 모놀리식**(이슈 #48) |
| 8 | **이식성** Portability | **3.0** | bare RN·fastlane 배포 | 단일 플랫폼(Android/web 없음) |

**종합**: 초기 베타치고 **견고(중상, ~3.2/5)**. 명확한 4대 약점 →
**① 접근성 0 · ② 저장 보안 미구현 · ③ 성능/크래시 실측 부재 · ④ UI 모놀리식.**
(ISO 25010:2023의 추가 특성 **Safety**: 마감/예식 알림이 안전 관련 — 알림 신뢰성 中.)

## Apple 심사·성능 준비도 체크
| 항목 | 상태 |
|---|---|
| Review Guidelines(개인정보·기능완결·크래시) | 대체로 OK, 단 로그인 필수화 금지 등 확인 |
| HIG(아이콘·레이아웃·다크모드) | 아이콘/세이프에어리어 개선됨, 다크모드 지원 |
| privacy manifest·암호화 선언 | 완료 |
| **접근성** | ❌ 미대응(심사 리스크 낮으나 품질·리뷰 평판 영향) |
| **성능 실측**(MetricKit) | ❌ 미도입 |
| **크래시 리포팅** | ❌ 없음 |

## 보안 (OWASP MASVS 요약)
- MASVS-STORAGE: 민감 PII 로컬 **평문** → 미흡(암호화/보호 필요)
- MASVS-AUTH: 앱잠금/생체 **OFF** → 미흡
- MASVS-NETWORK: ATS on, 로컬 우선 → 양호
- MASVS-PRIVACY: 추적 없음·마스킹 → 양호
- 판정: **L1 부분 충족** — 저장 보호·앱잠금이 최대 공백.

## 갭 → 조치 매핑 (핵심 의사결정)
> **원칙: 품질·컴플라이언스 결함은 "전원 무료"로 고친다. 유료(Pro)는 "이미 좋은데 더
> 많이/깊이/안전하게"인 것만.** 접근성·기본 보안·성능을 유료로 묶는 건 안티패턴(심사·신뢰 훼손).

### 🟢 무료로 반드시 보완 (베이스라인/컴플라이언스)
- **접근성**(VoiceOver 라벨·Dynamic Type) — Apple 표준·품질 기본선
- **성능 실측**(MetricKit)·**크래시 리포팅** — 내부 품질/안정성
- **기본 앱잠금·생체 잠금** — 민감정보 앱의 최소 안전선(전원 제공 권장)
- `index.tsx` 모듈 분해(이슈 #48) — 내부 유지보수성
- OCR 정확도 상향 — 핵심 가치(무료도 좋아야 입소문)

### 💎 Pro로 묶기 좋음 (규모·깊이·프리미엄 안전)
- **클라우드 백업·동기화 + 백업 암호화** — 데이터 안전 프리미엄(보안 갭을 "프리미엄"으로 전환)
- **다중 차량 · 상세 수익 리포트 · CSV 내보내기** — 규모·깊이
- **시간창/정밀 동선(지오코딩)** — 고급 자동화(성능·호환성 갭을 Pro 가치로)
- **OCR 클라우드 폴백 우선 처리** — 정확도 프리미엄

### 정리
- **저장 보안**: 기본 잠금은 무료, **암호화된 클라우드 백업**은 Pro → 갭을 무료보완+유료가치로 동시 해결.
- **OCR 정확도**: 무료로 기본 상향, **클라우드 폴백 우선권**은 Pro.
- **동선**: 기본 정렬 무료, **시간창·정밀 동선**은 Pro.
