# 4.2 목표 로드맵 & 재채점

`QUALITY_ASSESSMENT.md`(ISO/IEC 25010 기준) 후속. "Pro로 묶을 부분은 4.2급이어야
한다"는 목표 아래, 1차 보강을 실제 구현하고 다시 채점한다. (점수는 5점 척도,
근거는 코드·테스트 실측)

## 4.2 정의
3.0=동작 · 4.0=견고+계측+엣지 · **4.2=우수(자동화·검증·프리미엄 완성)** · 5.0=업계 최고

**종합: ~3.2 → ~3.4/5** (라운드 1~3 누적). 목표 4대 약점 중 **기능 적합성 4.2 도달**
(상세 리포트·다중차량 완료), **신뢰성 4.0**, 사용성·보안은 큰 폭 상승(각 +0.7)했으나
**아직 4.2 미도달**. 4.2 완성은 아래 심화(사용성·보안·성능·구조)가 필요.

## 4.2 도달까지 남은 심화 (특성별 "얼마나 더")
- **사용성 3.5→4.2**: 전 화면 접근성 감사 + 전면 Dynamic Type 채택(가변 레이아웃) + 대비 자동검증 [L]
- **보안 3.2→4.2**: 민감필드 at-rest 암호화 + 생체(Face ID) + **암호화 클라우드 백업(Pro)** [L]
- **성능 3.0→4.2**: MetricKit 성능 예산(콜드런치<400ms·hang율) + 리스트 가상화 [M]
- **기능 4.2 도달**: 상세 리포트·다중차량 완료. 남은 축은 OCR 정확도(84→90%+)로,
  기능 게이트가 아니라 텔레메트리로 상시 개선하는 모델 품질 축으로 분리한다. [상시]
- **호환성 3.0→4.2**: 지오코딩+GPS → 정밀 동선(Pro) [M]
- **유지보수 3.5→4.2**: index.tsx(5,138줄) 화면 단위 분해 [L]

## Pro 묶기 관점 결론
- **CSV 내보내기**: 실기능화 → **4.0급, Pro로 묶을 준비 됨.**
- **상세 수익 리포트**: 이번 라운드 실기능화(기간 비교·분류/지역/거래처 분해·건당/활동일
  평균·순이익률·완료율·최고 순익일) → **4.1급, Pro 게이트 적용 완료.**
- **다중 차량 관리**: 이번 라운드 실기능화(등록 차량 레지스트리·폼 칩 선택·차량별
  비용/연비/비중 분해) → **4.2급, Pro 게이트 적용 완료.** 기능 적합성 4.2 도달.
- **암호화 백업 / 정밀 동선**: 4.2로 올리려면 심화 필요 → "빌드해서 4.2 만든 뒤 Pro"
  원칙대로 순차 진행.
- **접근성·기본보안·성능·구조**: 무료로 계속 끌어올려 심사·신뢰 방어선 확보.
